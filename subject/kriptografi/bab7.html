Pengantar Kriptografi

🔐 Definisi Kriptografi

Kriptografi adalah ilmu yang mempelajari teknik pengamanan informasi dengan cara mengubah pesan asli (plaintext) menjadi bentuk yang tidak dapat dibaca (ciphertext), sehingga hanya pihak yang berwenang yang dapat mengembalikannya ke bentuk semula.

Kriptografi berperan penting dalam menjaga keamanan data di era digital 🌐, terutama untuk melindungi kerahasiaan, integritas, dan autentikasi informasi.

🧠 Konsep Dasar Kriptografi
  • 🔑 Plaintext: pesan asli sebelum proses enkripsi
  • 🔒 Ciphertext: pesan yang telah dienkripsi
  • ⚙️ Enkripsi: proses mengubah plaintext menjadi ciphertext
  • 🔓 Dekripsi: proses mengembalikan ciphertext menjadi plaintext
  • 🗝️ Kunci (Key): informasi yang digunakan dalam proses enkripsi dan dekripsi

Sebagai contoh, dalam komunikasi digital 💬, pesan yang dikirim biasanya diamankan dengan teknik enkripsi agar tidak dapat dibaca oleh pihak yang tidak berwenang.

⚡ Penerapan Kriptografi
  • 🏦 Keamanan transaksi perbankan dan e-commerce
  • 📧 Keamanan komunikasi email dan pesan instan
  • 🌍 Pengamanan website melalui HTTPS
  • 🔐 Sistem autentikasi dan login pengguna
📌 Catatan Penting

Kriptografi modern menggunakan algoritma seperti AES dan RSA yang dirancang agar sangat sulit dipecahkan tanpa kunci yang benar.

Modular Arithmetic Operations

Misalkan a dan b adalah bilangan bulat, serta m adalah bilangan bulat positif (modulus). Operasi aritmetika modular merupakan operasi hitung yang dilakukan dalam sistem modulo m, yaitu hasilnya selalu direduksi ke dalam rentang 0 sampai m − 1.

➕
Penjumlahan Modulo (a + b) mod m = ((a mod m) + (b mod m)) mod m
➖
Pengurangan Modulo (a − b) mod m = ((a mod m) − (b mod m)) mod m
✖️
Perkalian Modulo (a × b) mod m = ((a mod m) × (b mod m)) mod m
🔁
Sifat Kompatibilitas Operasi modulo kompatibel dengan operasi aritmetika dasar: hasil selalu dapat direduksi kembali ke mod m
🧮
Invers Aritmetika Modular a⁻¹ mod m ada jika dan hanya jika gcd(a, m) = 1
🔐
Peran dalam Kriptografi Operasi modular menjadi dasar algoritma seperti Caesar Cipher, RSA, dan Diffie-Hellman
a ≡ b (mod m) ⇔ m membagi (a − b)
💡 Intuisi Modular Arithmetic: Aritmetika modular dapat dibayangkan seperti jam ⏰. Setelah melewati angka maksimum (m), hitungan kembali ke awal (0), sehingga membentuk siklus berulang.

Caesar Cipher

Caesar Cipher adalah salah satu teknik kriptografi klasik yang digunakan untuk mengamankan pesan dengan cara menggeser setiap huruf dalam plaintext berdasarkan nilai kunci tertentu (K).

🔐 Definisi Caesar Cipher

Caesar Cipher merupakan metode substitusi sederhana di mana setiap huruf pada plaintext digantikan oleh huruf lain dengan pergeseran tetap sebanyak K posisi dalam alfabet.

🧠 Konsep Dasar
  • Plaintext: pesan asli sebelum enkripsi
  • Ciphertext: pesan hasil enkripsi
  • Key (K): jumlah pergeseran huruf
  • Enkripsi: proses mengubah plaintext menjadi ciphertext
  • Dekripsi: proses mengembalikan ciphertext menjadi plaintext
🧮 Rumus Matematis

Enkripsi: C = (P + K) mod 26
Dekripsi: P = (C − K) mod 26
Dengan ketentuan: A = 0, B = 1, ..., Z = 25

🔤 Representasi Alfabet

abcdefghijklm
0123456789101112
nopqrstuvwxyz
13141516171819202122232425
📌 Contoh Enkripsi

Plaintext: SAKAY
Key: K = 4

Hasil: WECEO

📌 Contoh Enkripsi Lengkap

Plaintext: RUTH DAN FEBRI
Key: K = 3

R → U, U → X, T → W, H → K
D → G, A → D, N → Q
F → I, E → H, B → E, R → U, I → L

Hasil Ciphertext: UXWK GDQ IHEUL
🔓 Dekripsi

Dekripsi dilakukan dengan rumus: P = (C − K) mod 26, sehingga ciphertext dapat dikembalikan ke plaintext asli.

⚡ Kelebihan & Kelemahan

Kelebihan: mudah dipahami dan diimplementasikan
Kelemahan: sangat mudah dipecahkan dengan brute force karena hanya memiliki 25 kemungkinan kunci

Playfair Cipher

Playfair Cipher merupakan salah satu metode kriptografi klasik yang dikembangkan oleh Charles Wheatstone (1854) dan dipopulerkan oleh Lord Playfair. Algoritma ini menggunakan matriks 5×5 untuk melakukan enkripsi berdasarkan pasangan huruf (digraf).

🔐 Konsep Dasar

Playfair Cipher adalah teknik enkripsi yang menggantikan huruf dalam bentuk pasangan (digraf), bukan satu huruf tunggal seperti Caesar Cipher. Huruf I dan J dianggap sama untuk menyesuaikan ukuran matriks 5×5.

⚙️ Aturan Enkripsi
  • ➡️ Jika dua huruf berada pada baris yang sama → geser ke kanan
  • ⬇️ Jika dua huruf berada pada kolom yang sama → geser ke bawah
  • 🔲 Jika berbeda baris & kolom → bentuk persegi panjang dan ambil sudut berlawanan
  • ➕ Jika ada huruf sama dalam satu pasangan → tambahkan huruf X

GTPIB
CDFHK
LOQRU
VWXYZ
📘 Contoh Enkripsi 1

Plaintext: FOR A BETTER FUTURE
Digraf: FO RA BE TX ER FU TU RE

Hasil Ciphertext: DQ YI TN PW AO KQ BO OA

🔤 Matriks Playfair Cipher (Key: KELOMPOK)

📘 Contoh Enkripsi 2

Plaintext: BELAJAR

Enkripsi:

Hasil Ciphertext: LAEBGCSW

⚡ MATERI TAMBAHAN
📊 Kelebihan
  • Lebih aman dibanding Caesar Cipher
  • Menghilangkan pola frekuensi huruf tunggal
  • Menggunakan pasangan huruf sehingga lebih kompleks
⚠️ Kelemahan
  • Masih bisa dipecahkan dengan analisis frekuensi digraf
  • Tidak aman untuk kriptografi modern
  • Bergantung pada kerahasiaan keyword
💡 Intuisi Sederhana

Playfair Cipher dapat dianggap sebagai “Caesar Cipher versi pasangan huruf (digraf)”, karena tidak lagi memproses satu huruf, tetapi dua huruf sekaligus.

RSA Encryption

RSA (Rivest–Shamir–Adleman) adalah algoritma kriptografi kunci publik yang digunakan untuk mengamankan data dalam komunikasi digital. RSA banyak digunakan dalam sistem keamanan seperti HTTPS, tanda tangan digital, dan enkripsi data.

🔐 Pengertian RSA

RSA adalah sistem kriptografi asimetris yang menggunakan dua kunci: public key untuk enkripsi dan private key untuk dekripsi. Keamanan RSA bergantung pada kesulitan faktorisasi bilangan besar menjadi bilangan prima.

🧠 Konsep Dasar RSA
  • 🔓 Public Key (e, n): digunakan untuk enkripsi
  • 🔒 Private Key (d, n): digunakan untuk dekripsi
  • 🔢 n = p × q (dua bilangan prima besar)
  • φ(n) = (p − 1)(q − 1)
  • e dan d saling invers modulo φ(n)
⚙️ Langkah Pembentukan Kunci RSA
  1. Pilih dua bilangan prima p dan q
  2. Hitung n = p × q
  3. Hitung φ(n) = (p−1)(q−1)
  4. Pilih e sehingga gcd(e, φ(n)) = 1
  5. Hitung d sehingga (d × e) mod φ(n) = 1
🔢 Contoh RSA Sederhana
📘 Diketahui

p = 61, q = 53
n = 61 × 53 = 3233
φ(n) = 60 × 52 = 3120
e = 17

🔑 Kunci yang Dihasilkan

Public Key = (e = 17, n = 3233)
Private Key = (d = 2753, n = 3233)

🔐 Proses Enkripsi
📤 Enkripsi RSA

Rumus enkripsi: C = P^e mod n

Misal plaintext P = 65

C = 65^17 mod 3233 = 2790

Hasil ciphertext: 2790

🔓 Proses Dekripsi
📥 Dekripsi RSA

Rumus dekripsi: P = C^d mod n

P = 2790^2753 mod 3233 = 65

Hasil kembali ke plaintext: 65

📊 Tabel Istilah RSA
Istilah Penjelasan Rumus / Contoh
Public Key Kunci yang digunakan untuk enkripsi (e, n)
Private Key Kunci untuk dekripsi (d, n)
Modulus Hasil perkalian dua bilangan prima n = p × q
Totient Euler Jumlah bilangan relatif prima terhadap n φ(n) = (p−1)(q−1)
Enkripsi Mengubah plaintext menjadi ciphertext C = P^e mod n
Dekripsi Mengembalikan ciphertext menjadi plaintext P = C^d mod n
⚡ Keunggulan & Kelemahan RSA
📈 Keunggulan
  • Sangat aman untuk komunikasi digital
  • Tidak perlu membagikan private key
  • Digunakan di HTTPS, SSL, dan tanda tangan digital
📉 Kelemahan
  • Proses komputasi lebih lambat dibanding algoritma simetris
  • Membutuhkan bilangan besar untuk keamanan tinggi
💡 Intuisi RSA

RSA seperti kotak kunci: siapa pun bisa mengunci (encrypt) menggunakan public key, tetapi hanya pemilik private key yang bisa membukanya (decrypt).

Advanced Encryption Standard (AES)

🔐 Definisi AES

Advanced Encryption Standard (AES) adalah algoritma kriptografi simetris modern yang digunakan untuk mengenkripsi data secara aman dengan menggunakan satu kunci yang sama untuk proses enkripsi dan dekripsi.

AES merupakan standar enkripsi yang ditetapkan oleh NIST (National Institute of Standards and Technology) dan digunakan secara luas dalam keamanan data seperti komunikasi internet, aplikasi perbankan, dan penyimpanan data terenkripsi.

🧠 Konsep Dasar AES
  • 🔑 Menggunakan kunci simetris (enkripsi dan dekripsi memakai kunci yang sama)
  • 📦 Mengolah data dalam blok berukuran tetap (128-bit)
  • 🔁 Menggunakan beberapa ronde transformasi (10, 12, atau 14 ronde)
  • 🧩 Operasi berbasis substitusi dan permutasi
⚙️ Parameter AES
  • AES-128 → kunci 128-bit (10 ronde)
  • AES-192 → kunci 192-bit (12 ronde)
  • AES-256 → kunci 256-bit (14 ronde)
🔄 Tahapan Proses AES

Proses utama AES terdiri dari beberapa langkah:

🔑
Key Expansion Menghasilkan round key dari kunci utama untuk setiap ronde.
🧱
SubBytes Mengganti setiap byte menggunakan S-Box (substitusi non-linear).
🔀
ShiftRows Menggeser baris-baris dalam state array untuk difusi data.
🔄
MixColumns Mencampur kolom untuk meningkatkan keamanan (tidak pada ronde terakhir).
➕
AddRoundKey Menggabungkan state dengan round key menggunakan operasi XOR.
🔐 Proses Enkripsi AES
📤 Tahapan Enkripsi
  1. Initial Round → AddRoundKey
  2. Ronde utama: SubBytes → ShiftRows → MixColumns → AddRoundKey
  3. Ronde terakhir: SubBytes → ShiftRows → AddRoundKey (tanpa MixColumns)
🔓 Proses Dekripsi AES
📥 Tahapan Dekripsi

Dekripsi AES merupakan kebalikan dari proses enkripsi dengan urutan:

  • InvShiftRows
  • InvSubBytes
  • AddRoundKey
  • InvMixColumns
⚡ Aplikasi AES di Dunia Nyata
🌐
HTTPS Digunakan untuk mengamankan komunikasi web.
📱
Aplikasi Chat WhatsApp, Telegram menggunakan AES untuk enkripsi pesan.
🏦
Perbankan Melindungi transaksi digital dan data nasabah.
💡 Intuisi AES

AES dapat dianalogikan seperti proses “mengacak data berlapis-lapis”. Setiap ronde membuat data semakin sulit ditebak sampai hampir mustahil diretas tanpa kunci.

Vigenère Cipher

🔐 Definisi Vigenère Cipher

Vigenère Cipher adalah metode kriptografi klasik yang menggunakan kunci berulang (repeating key) untuk melakukan enkripsi dengan cara menggeser huruf plaintext berdasarkan huruf pada kunci.

Berbeda dengan Caesar Cipher yang hanya menggunakan satu nilai pergeseran, Vigenère Cipher menggunakan beberapa nilai pergeseran yang berubah-ubah sesuai kata kunci, sehingga lebih sulit untuk dipecahkan dengan analisis frekuensi sederhana.

🧠 Konsep Dasar

🔑
Plaintext Pesan asli yang akan dienkripsi.
🧩
Key (Kunci) Kata kunci yang diulang hingga sepanjang plaintext.
🔐
Ciphertext Hasil pesan yang sudah dienkripsi.

🧮 Rumus Matematis

Enkripsi: C = (P + K) mod 26 Dekripsi: P = (C − K) mod 26 A = 0, B = 1, ..., Z = 25

🔤 Tabel Alfabet

abcdefghijklm
0123456789101112
nopqrstuvwxyz
13141516171819202122232425

📌 Cara Kerja Vigenère Cipher

⚙️ Langkah Enkripsi
  1. Tentukan plaintext
  2. Tentukan keyword
  3. Ulangi keyword hingga sepanjang plaintext
  4. Setiap huruf plaintext digeser sesuai huruf keyword

📘 Contoh Enkripsi

🔐 Contoh Kasus

Plaintext: ATTACKATDAWN
Key: LEMON

Key diperpanjang: LEMONLEMONLE A + L = L T + E = X T + M = F A + O = O C + N = P K + L = V A + E = E T + M = F D + O = R A + N = N W + L = H N + E = R Ciphertext: LXFOPVEFRNHR

🔓 Dekripsi

📥 Proses Dekripsi

Dekripsi dilakukan dengan mengurangi setiap huruf ciphertext dengan huruf pada key:

P = (C − K) mod 26

Hasil dekripsi akan mengembalikan ciphertext menjadi plaintext asli.

⚡ Kelebihan & Kelemahan

📈
Kelebihan Lebih aman dibanding Caesar Cipher karena menggunakan banyak pergeseran.
📉
Kelemahan Masih bisa dipecahkan menggunakan analisis Kasiski dan frekuensi lanjutan.
💡 Intuisi Vigenère Cipher: Vigenère Cipher dapat dianggap sebagai “Caesar Cipher dengan kunci yang berubah-ubah”, sehingga lebih sulit ditebak karena pola pergeseran tidak konstan.

Hash Functions & Message Digest

🔐 Definisi Hash Function

Hash function adalah fungsi kriptografi yang mengubah data berukuran sembarang menjadi nilai tetap (fixed-size) yang disebut hash value atau message digest.

Hash function digunakan untuk memastikan integritas data, yaitu memastikan bahwa data tidak berubah selama proses transmisi atau penyimpanan. Jika satu karakter saja berubah, maka hasil hash akan berbeda total.

🧠 Konsep Dasar Hash Function

📦
Input Data berukuran bebas (teks, file, password).
🔁
Proses Hashing Transformasi satu arah (tidak bisa dibalik).
🔐
Output Nilai tetap (message digest / hash).

⚙️ Sifat Hash Function

📌 Karakteristik Utama
  • 🔒 One-way → tidak bisa dikembalikan ke input asli
  • 📏 Fixed length → output selalu sama panjang
  • ⚡ Deterministik → input sama menghasilkan hash sama
  • 💥 Avalanche effect → perubahan kecil mengubah seluruh hash
📊 Contoh Hash Functions Populer
Algoritma Panjang Output Status Keamanan
MD5 128-bit ❌ Tidak aman (sudah rentan collision)
SHA-1 160-bit ⚠️ Mulai tidak direkomendasikan
SHA-256 256-bit ✅ Aman dan banyak digunakan

🔐 Message Digest

📌 Definisi Message Digest

Message digest adalah hasil akhir dari proses hash function, yaitu representasi unik dari suatu data.

🧮 Proses Hashing

Input Data → Hash Function → Message Digest
Contoh: "HELLO" → SHA-256 → 2cf24dba5fb0a... (hasil hash panjang tetap)

🔐 Fungsi Hash dalam Keamanan

🛡️
Verifikasi Data Mengecek apakah file berubah atau tidak.
🔑
Password Storage Password tidak disimpan langsung, tetapi dalam bentuk hash.
📡
Digital Signature Menjamin keaslian dan integritas pesan.
💡 Intuisi Hash Function: Hash function seperti “sidik jari digital” — setiap data memiliki identitas unik yang tidak bisa dibalik ke bentuk aslinya.
